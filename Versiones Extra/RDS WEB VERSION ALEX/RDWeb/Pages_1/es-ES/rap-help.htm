¿Qué es Conexión de RemoteApp y Escritorio?

Conexión de RemoteApp y Escritorio ofrece una vista personalizada de los programas RemoteApp y los escritorios virtuales que el administrador de red ha puesto a su disposición.

¿Qué es RemoteApp?

RemoteApp permite obtener acceso a programas de un equipo remoto a través de Servicios de Escritorio remoto. Aunque los programas se ejecuten en un equipo remoto, los programas RemoteApp se comportan como si se ejecutaran en el equipo local. Por ejemplo, un programa RemoteApp tiene su propia entrada en la barra de tareas y puede cambiar de tamaño, minimizar o maximizar la ventana del programa.

Inicio de un programa RemoteApp

Para iniciar un programa RemoteApp o una sesión de Escritorio remoto, haga clic en el icono de programa en Acceso web de RD. Cuando se le soliciten credenciales de usuario, inicie sesión con el nombre de usuario y la contraseña de red.

¿Qué es la pestaña Escritorio remoto?

La ficha Escritorio remoto permite conectarse desde el sitio de Acceso web de RD a cualquier escritorio que esté configurado para aceptar conexiones de Escritorio remoto.

Configuración pública frente a configuración privada del equipo

Si se conecta al sitio de Acceso web de RD desde un equipo público, como un quiosco multimedia en un establecimiento público, o desde un equipo que comparte con otros usuarios, haga clic en Éste es un equipo público o compartido. Deberá proporcionar su nombre de usuario y contraseña cada vez que inicie sesión en el sitio de Acceso web de RD.

Si está usando un equipo de trabajo asignado a usted, que no comparte con otras personas, haga clic en Éste es un equipo privado. Se recordará su nombre de usuario, así solo tendrá que proporcionar su contraseña cada vez que inicie sesión en el sitio de Acceso web de RD.

Para proteger contra acceso no autorizado, las sesiones de Acceso web de RD finalizan automáticamente tras un período de inactividad. Si finaliza su sesión de Acceso web de RD, deberá iniciar sesión de nuevo. El tiempo que transcurre antes de finalizar una sesión es determinado por el administrador.

Requisitos del equipo

Para usar Acceso web de RD, el equipo debe ejecutar al menos Internet Explorer 6.0 y una versión de Conexión a Escritorio remoto (RDC) que admita al menos el Protocolo de escritorio remoto (RDP) 6.1.

Para determinar la versión de RDP que RDC admite, inicie Conexión a Escritorio remoto, haga clic en el icono de la esquina superior izquierda del cuadro de diálogo Conexión a Escritorio remoto y elija Acerca de. La versión de RDP que RDC admite aparece en el cuadro de diálogo Acerca de Conexión a Escritorio remoto.

RDC 6.1 admite el Protocolo de escritorio remoto 6.1, que se incluye con los siguientes sistemas operativos:

  • Windows Server 2008
  • Windows Vista con SP1
  • Microsoft Windows XP con SP 3
The version of RDC in Windows 7 and Windows Server 2008 R2 supports RDP 7.0.

Additionally, the Remote Desktop Services ActiveX Client control must be enabled. The ActiveX control is included with RDC 6.1 and the version of RDC in Windows 7 and Windows Server 2008 R2.

The version of RDC that you're using determines which features of RD Web Access are available to you.

For more information about client requirements for using RD Web Access, see http://go.microsoft.com/fwlink/?LinkId=142240.

Se me pide que ejecute el control de cliente ActiveX de Servicios de Escritorio remoto. ¿Cómo lo hago?

Si se le pide que ejecute el control de cliente ActiveX de Servicios de Escritorio remoto al tener acceso a Acceso web de RD, siga alguno de estos procedimientos, según el sistema operativo:

  • Si ejecuta Windows Server 2003 o Windows XP, y se le pide que ejecute el control de cliente ActiveX de Servicios de Escritorio remoto, haga clic en la línea de mensaje, elija Ejecutar control ActiveX y haga clic en Ejecutar. Una vez habilitado el control, actualice la página web.
  • Si ejecuta Windows Server 2008 o Windows Vista, haga clic en el mensaje de advertencia en la barra de información de Internet Explorer, elija Complemento deshabilitado y haga clic en Ejecutar control ActiveX. Al hacer esto, puede que se muestre una advertencia de seguridad. Asegúrese de que el editor del control ActiveX sea "Microsoft Corporation" antes de hacer clic en Ejecutar.
  • Si ejecuta Windows Server 2008 R2 o Windows 7, además de los pasos anteriores, tiene la opción de permitir que el control ActiveX se ejecute solo en este sitio web.
If the Internet Explorer Information bar doesn't appear, you can enable the ActiveX control by using the Manage Add-ons tool on the Tools menu of Internet Explorer.

If your computer meets all of the client requirements but you do not see any programs listed, contact your network administrator or the webmaster of the website.
